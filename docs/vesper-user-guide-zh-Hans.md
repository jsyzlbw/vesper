# Vesper 用户说明书

> 当前版本：开发预览版
> 适用范围：iPhone Companion App

## 1. Vesper 是什么

Vesper 是一个对话式私人助理。你可以直接用自然语言描述计划，Vesper 会在信息足够时生成一张可编辑的提醒卡片。只有你点击确认后，应用才会创建通知、真闹钟或日历事件。

当前版本重点支持“自然语言安排提醒”，不需要 Mac 常驻服务。

## 2. 初次设置

进入“设置”，点击右上角“新增 Provider”：

1. 选择平台。
2. 填写显示名称、Base URL、模型名称和 API Key。
3. 查看“实际 Endpoint”，确认应用最终请求的地址。
4. 点击“测试连接”。
5. 测试成功后保存，并保持 Provider 启用。

API Key 保存在系统 Keychain 中。界面不会明文展示已保存的 Key。

### DeepSeek 示例

- Base URL：`https://api.deepseek.com`
- 实际 Endpoint：`https://api.deepseek.com/chat/completions`
- 不需要填写 `https://api.deepseek.com/anthropic`

## 3. 当前 Provider 支持情况

| 平台 | 可在设置中选择 | 当前可用于聊天 |
| --- | --- | --- |
| OpenAI | 是 | 是 |
| DeepSeek | 是 | 是 |
| 硅基流动 | 是 | 是 |
| Custom | 是 | 是，需兼容 OpenAI Chat Completions 与 SSE 流式响应 |
| Anthropic | 是 | 暂未接入聊天流 |
| Gemini | 是 | 暂未接入聊天流 |

Anthropic 和 Gemini 已具备配置入口与请求构造基础，但当前聊天界面只选择 OpenAI 兼容 Provider。后续版本会补齐各自的流式协议。

## 4. 使用自然语言创建提醒

在“对话”页直接描述需求。例如：

```text
从下周日开始，每两周周日下午 3 点学习两小时，提前 30 分钟提醒我。
```
如果信息不完整，Vesper 会先追问，例如活动持续时间、开始日期或提醒方式。

信息完整后，Vesper 会生成确认卡片。卡片中可看到：

- 安排方式：固定时间或自动安排到空闲时间
- 事件时间
- 重复规则
- 事件持续时间
- 系统通知是否开启
- 通知实际触发时间
- 真闹钟是否开启
- 真闹钟实际触发时间
- 是否同步到系统日历

点击卡片中的任意条目可进入编辑页。保存修改后，再点击“确认创建”。

## 5. 通知、真闹钟和日历

### 系统通知

普通“提醒”默认使用系统通知。通知支持声音与震动，可以设置相对事件时间提前多少分钟触发。

例如：

```text
每两周周日下午 3 点学习，提前 30 分钟提醒我。
```

对应的事件时间是 `15:00`，通知时间是 `14:30`。

### 真闹钟

只有你明确说“闹钟”或 `alarm` 时，Vesper 才会默认开启真闹钟。普通提醒不会擅自创建真闹钟。

真闹钟使用 Apple AlarmKit，行为接近系统闹钟。它仅支持 **iOS 26 及以上**。在较旧系统上，Vesper 会说明系统版本限制，不会悄悄降级成普通通知。

编辑卡片时，你也可以手动开启或关闭真闹钟，并独立设置闹钟提前量。

### 日历

开启“同步到日历”后，Vesper 会在你确认时请求日历权限，并创建事件。重复提醒会按规则创建重复日程。

## 6. 自动安排空闲时间

你可以让 Vesper 自动安排时间，例如：

```text
帮我今天安排 45 分钟复习，放进空闲时间，晚上 10 点前完成。
```

Vesper 会读取所有可见日历，在指定时间范围内选择最早可用时段，然后生成卡片等待确认。

## 7. 重复规则

当前支持：

- 单次
- 每天，支持间隔天数
- 每周，支持多个星期与间隔周数
- 每月指定日期
- 每月最后一天
- 每年指定日期
- 限制重复次数

应用会持续补充未来通知和真闹钟，避免长期重复计划只创建最初一小段。

## 8. 语言

进入“设置”中的“App 语言”，可选择：

- 跟随系统
- 简体中文
- English

即使 App 使用英文，如果你用中文说话，AI 也会优先用中文回复。

## 9. 权限与确认机制

当前默认策略是“执行前确认”：

- AI 可以提出建议。
- 卡片生成后仍可编辑。
- 只有点击确认后，应用才会创建通知、真闹钟或日历事件。
- 取消提醒时，会清理已创建的系统资源。
- 如果创建过程被中断，卡片会提供恢复入口。

## 10. 当前限制

这是开发预览版，目前仍有以下限制：

- Anthropic 和 Gemini 尚未接入聊天流。
- OAuth 登录尚未实现，目前使用 API Key。
- iCloud 同步尚未实现，数据目前保存在本机。
- “时间线”和“审计”页已有入口，但完整内容展示仍待完善。
- 通用 AI 命令执行框架已有核心基础，但尚未完整接入聊天页。
- iOS 模拟器中的中文键盘存在系统级输入异常；尚未在真机验证。可暂时使用复制粘贴或英文输入进行测试。
- AI 输出仍可能偶尔不生成结构化卡片。重新描述需求通常可以恢复，后续版本会继续增强稳定性。

## 11. 建议测试用例

普通通知：

```text
从下周日开始，每两周周日下午 3 点学习两小时，提前 30 分钟提醒我。
```

真闹钟：

```text
明天早上 7 点设置闹钟提醒我起床。
```

通知与闹钟同时开启：

```text
明天早上 7 点提醒我起床，并且设置闹钟。
```

自动排期：

```text
帮我今天安排 45 分钟读书，放进空闲时间，晚上 10 点前完成。
```
